Overview

HTML5 Studio is a free, self-hosted animation studio for creating HTML5 display ads. Design keyframe animations in a visual timeline editor, manage every ad size as a separate scene within a single project, then export production-ready ZIP packages for Google Display Network, DV360, The Trade Desk, or any HTML5-compatible ad server.

Exports are fully self-contained — a single ZIP per size with zero runtime dependencies, platform-specific click tags, and built-in ad spec validation. No accounts, no subscriptions, no cloud required.

Quick Start

1. Open the app

When you first open HTML5 Studio, you'll see a pre-built Meridian Bank demo project with five common ad sizes. This is a great starting point for exploring the interface.

2. Choose a scene

Click a tab in the Scene Tabs bar to switch between ad sizes. Each scene is an independent canvas with its own layers, animation, and dimensions.

3. Edit layers

Click any element on the canvas to select it. Use the Properties Panel on the right to change its position, size, color, text content, and more. Drag handles on the canvas to move and resize layers visually.

4. Animate

Move the playhead in the Timeline to a new frame, then change a layer's properties. Press A to add a keyframe. The layer will now animate between its start and end states. Press Space to preview.

5. Export

Click Export HTML5 in the top bar. Choose your target platform in the export settings dropdown, then download the ZIP. Each export is a standalone HTML5 package ready for upload to an ad server.

Interface Tour

The interface is divided into five main areas:

Top Bar

The toolbar across the top of the screen. From left to right:

  • Upload SVG/AI — import vector files as layers
  • Project Manager — load saved projects
  • Save Project — save to the server's filesystem
  • Preview — preview your ad in-browser before exporting
  • Export HTML5 — export the current scene as a ZIP package
  • Export Settings (gear icon) — set filename and target platform
  • Batch Export (archive icon) — export all scenes at once
  • Undo / Redo — step through edit history
  • Help — open this manual or view keyboard shortcuts
  • Theme Toggle — switch between dark and light mode

Scene Tabs

A row of tabs below the top bar, one per scene. Click to switch scenes. Double-click a tab to rename it. Right-click for a context menu (rename, delete). Drag tabs to reorder. Click the + button to add a new scene.

Canvas Area

The central editing area where your ad is rendered. Above it is a toolbar with:

  • Grid — toggle a grid overlay
  • Snap to Grid — snap layers to grid lines while dragging
  • Smart Guides — show alignment guides when moving layers near edges or centers
  • Zoom — zoom in/out, choose a preset, or fit to window

Properties Panel

The right-side panel has tabs for editing different aspects of the selected layer:

  • Layer — position, size, rotation, opacity, type-specific options
  • Animation — keyframes, easing, timing, loop settings
  • Stage — scene dimensions, duration, and FPS
  • Assets — image library shared across all scenes
  • Templates — save/load scene templates

Timeline

The bottom section showing the layer list (left) and keyframe timeline (right). Playback controls (play, pause, stop, frame navigation) sit above the timeline. Each layer row shows its keyframes as dots along the time ruler.

Scenes

A scene represents a single ad size (e.g., 300×250, 728×90). One project can contain many scenes, letting you manage an entire ad set in a single file.

  • Add a scene — click the + button in the scene tabs. You'll be prompted to choose a size from the built-in IAB templates or enter custom dimensions.
  • Rename — double-click the tab label, or right-click and select "Rename."
  • Delete — right-click the tab and select "Delete." You'll be asked to confirm.
  • Reorder — drag tabs left or right.

Each scene has its own independent layer stack, keyframe animation, dimensions, duration, and FPS.

Layers

Layers are the building blocks of your ad. Each layer sits in a stacking order — layers higher in the list appear in front.

Layer Types

TypeDescription
ColorA solid-color rectangle. Great for backgrounds and overlays.
GradientA linear or radial gradient fill. Supports multiple color stops.
TextEditable text with font, size, weight, alignment, color, and effects (shadow, stroke).
ImageAn uploaded raster image with fit options (contain, cover, fill).
GroupA container for other layers. Groups can be looped as a unit.

Layer Operations

  • Add — right-click in the layer list to add a new layer of any type.
  • Select — click a layer in the list or on the canvas.
  • Reorder — drag layers up or down in the layer list.
  • Duplicate — Ctrl+D or right-click > Duplicate.
  • Copy / Paste — Ctrl+C to copy, Ctrl+V to paste in the current scene, Ctrl+Shift+V to paste into all scenes.
  • Delete — Delete key or right-click > Delete.
  • Toggle Visibility — click the eye icon or press V.
  • Lock — click the lock icon to prevent accidental edits.

Canvas & Editing

The canvas shows your ad at the current zoom level. Click a layer to select it. Selected layers show resize handles and a rotation control.

  • Move — drag anywhere inside the selected layer.
  • Resize — drag a corner or edge handle.
  • Rotate — drag the rotation handle, or type a value in the Properties Panel.
  • Zoom — use the zoom controls in the canvas toolbar, or scroll while holding Ctrl/Cmd.
  • Grid — toggle from the canvas toolbar. When snap-to-grid is enabled, layers will snap to grid lines as you drag.
  • Smart Guides — when enabled, temporary alignment lines appear as you drag a layer near the edges or center of other layers or the stage.

Keyframes

Animation in HTML5 Studio is keyframe-based. You define how a layer should look at specific points in time (keyframes), and the software interpolates smoothly between them.

Adding Keyframes

  1. Select a layer.
  2. Move the playhead to the desired frame using the timeline slider or frame navigation buttons.
  3. Adjust the layer's properties (position, size, opacity, rotation, etc.).
  4. Press A to add a keyframe at the current frame. You'll see a dot appear on the timeline.

You can also right-click on the timeline to add a keyframe, or use the Animation tab in the Properties Panel to type precise keyframe values.

Editing Keyframes

  • Select — click a keyframe dot on the timeline.
  • Move — drag a keyframe dot left or right to change its timing.
  • Delete — select a keyframe and press Delete, or right-click > Delete Keyframe.
  • Edit values — select a keyframe and use the Animation tab to adjust its property values and easing.

Animatable Properties

The following properties can be animated between keyframes:

  • Position (X, Y)
  • Size (Width, Height)
  • Scale (X, Y)
  • Rotation
  • Opacity
  • Font size (text layers)
  • Letter spacing (text layers)

Duration & FPS

Set the scene's total duration (in seconds) and frames per second in the Stage tab of the Properties Panel. A 30-second animation at 30 FPS has 900 frames. Higher FPS means smoother animation but more frames to work with.

Easing Functions

Easing controls how a layer accelerates and decelerates between keyframes. Select a keyframe, then choose an easing function from the dropdown in the Animation tab.

CategoryVariantsCharacter
LinearlinearConstant speed, no acceleration
Power 1–4in, out, inOutGentle to strong curves (higher = snappier)
Sinein, out, inOutSmooth, natural motion
Expoin, out, inOutDramatic acceleration / deceleration
Elasticin, out, inOutSpringy overshoot
Backin, out, inOutSlight overshoot then settle
Bouncein, out, inOutBouncing ball effect
Tip For most display ads, power2.out or power3.out gives a polished, professional feel. Use elastic and bounce sparingly — they can be distracting in small ad units.

Loops & Sub-Timelines

Global Loop

Toggle the global loop in the Animation tab to make the entire scene repeat continuously. Most ad networks require ads to stop after 30 seconds, so keep this in mind during export.

Per-Layer Loops

Individual layers (and groups) can have their own independent loops. Press L or click the loop toggle in the layer list. When enabled, you can define a loop window — a range of frames that repeats indefinitely while the rest of the timeline plays normally.

This is perfect for effects like a pulsing CTA button or a continuously rotating element while the rest of the ad plays through its narrative.

Example Your ad is 5 seconds long. The intro text fades in over 1 second and stays put. Meanwhile, the CTA button has a loop window from frame 60 to frame 90 — it pulses continuously from the 2-second mark onward.

Layer Properties

Select a layer to see its properties in the right panel. All layers share these common properties:

  • Name — a label for the layer (shown in the timeline list)
  • Position — X and Y coordinates (pixels from top-left corner of the stage)
  • Size — width and height in pixels
  • Rotation — degrees (0–360)
  • Opacity — 0% (invisible) to 100% (fully opaque)
  • Visibility — toggle whether the layer is rendered
  • Lock — prevent selection and editing on the canvas

Text

Text layers support:

  • Content — the text string to display
  • Font Family — choose from Google Fonts or system fonts
  • Font Size — in pixels
  • Font Weight — 100 (thin) through 900 (black)
  • Style — normal or italic
  • Letter Spacing — space between characters (px)
  • Line Height — vertical spacing between lines (px)
  • Alignment — left, center, or right
  • Fill Color — hex, RGB, or CMYK color picker
  • Text Effects:
    • Shadow — color, blur radius, offset X/Y, opacity
    • Stroke — color, width, opacity

Gradients

Gradient layers support two types:

  • Linear Gradient — defined by start and end points. The gradient transitions along a straight line between these two points.
  • Radial Gradient — defined by a center point and radius. The gradient radiates outward from the center.

Both types use color stops — you can add multiple stops to create complex multi-color gradients. Each stop has a color and a position (0% to 100%).

Images

Image layers let you upload raster images and control how they fill the layer bounds:

Fit ModeBehavior
ContainScale to fit entirely within the bounds, preserving aspect ratio (may show gaps)
CoverScale to cover the entire bounds, preserving aspect ratio (may crop edges)
FillStretch to exactly fill the bounds (may distort)
NoneDisplay at original size, centered

Blend Modes

Every layer has a blend mode that controls how it composites with layers below it. Available modes:

  • Normal (default)
  • Multiply, Screen, Overlay
  • Darken, Lighten
  • Color Dodge, Color Burn
  • Hard Light, Soft Light
  • Difference, Exclusion

SVG / AI Import

Click the Upload SVG/AI button in the top bar to import a vector file. The file is parsed into editable layers.

When importing, you'll be asked:

  • Merge into current scene — adds the imported layers on top of your existing layers.
  • Replace current scene — clears the current scene and replaces it with the imported content.
Tip For best results, flatten complex SVG effects (filters, masks, clip paths) in your vector editor before importing. Simple shapes, paths, and text import most reliably.

Asset Library

The Assets tab in the Properties Panel shows all unique images used across every scene in your project. Each thumbnail displays a usage count.

To reuse an image from the library, select a layer on the canvas, then click the + button on the desired asset thumbnail.

Exporting HTML5

Click Export HTML5 in the top bar to export the current scene. The export produces a ZIP file containing:

  • index.html — the ad markup
  • script.js — animation logic with embedded easing functions (no external libraries)
  • styles.css — layout and styling
  • Image assets (bundled in the ZIP)

The exported code has zero runtime dependencies. Easing functions are baked directly into the JavaScript. No GSAP, no jQuery, no external CDN calls.

Export Settings

Click the gear icon next to the Export button to configure:

  • Filename — the name of the output ZIP file (default: ad_package)
  • Target Platform — determines click tag format and validation rules (see Platform Targets)

Platform Targets

TargetClick TagMax SizeNotes
Generic HTML5 Standard clickTag 150 KB IAB standard. No platform dependencies.
Google Display Network (GDN) clickTag 150 KB Self-contained, no external requests. 40-file max. 30-second loop limit.
DV360 / Enabler Enabler.getUrl() 200 KB Includes Enabler.js script tag automatically.
The Trade Desk clickTAG (capital TAG) 150 KB Uses TTD's variable naming convention.

Batch Export

If your project has multiple scenes, click the Batch Export button (archive icon) to export all scenes at once. This creates a master ZIP containing individual ZIPs for each scene.

Batch export is only available when the project has more than one scene.

Ad Spec Validation

Before every export, HTML5 Studio runs pre-flight validation checks against the selected platform's specifications:

  • File size — total ZIP size vs. platform limit
  • File count — number of files in the package (GDN has a 40-file limit)
  • Duration — animation length vs. platform loop limits
  • External requests — flags any external resource dependencies (GDN requires fully self-contained ads)

Validation results are shown in the Export Progress modal. Warnings let you know about potential issues; errors will prevent the export from completing.

Saving & Loading

Saving a Project

  • Click Save Project As... to save a new project. Enter a name and click Save.
  • Once saved, the button changes to Save Project, which overwrites the existing save.
  • Use Ctrl+S (Cmd+S on Mac) to quick-save.
  • A timestamp (e.g., "Saved 2 minutes ago") appears in the top bar after saving.

Loading a Project

Click the Project Manager button in the top bar to see all saved projects. Click a project to load it. The current unsaved work will be replaced.

Data Storage

Projects are stored as directories on the server's local filesystem:

  • data/projects/{uuid}/project.json — project metadata and animation data
  • data/projects/{uuid}/assets/ — uploaded image files

To back up your projects, copy the data/ directory. In Docker, this is persisted via a named volume.

Tips & Best Practices

Design

  • Start with the most constrained size first (e.g., 320×50 mobile banner), then adapt the layout for larger sizes.
  • Keep text large and readable — display ads are often viewed at small sizes in cluttered pages.
  • Use high-contrast colors for CTA buttons so they stand out.
  • Limit your palette to 2–3 brand colors for visual consistency across sizes.

Animation

  • Keep animations under 15 seconds total for the best viewer engagement.
  • Use the first 2–3 seconds to grab attention — many viewers will not watch the full loop.
  • End every ad on the CTA and brand name so they're visible even if the animation is paused.
  • Avoid excessive motion. Subtle easing (power2.out) often works better than dramatic effects.
  • Use per-layer loops for repeating elements (pulsing CTA, rotating icon) while keeping the main narrative linear.

Export

  • Always check the validation results before uploading to an ad network.
  • Compress images before importing — the file size limit applies to the entire ZIP package.
  • Use batch export to quickly generate all sizes in one step.
  • Test exported ads in different browsers. The exported code uses standard CSS and JS, but always verify.

Project Management

  • Save frequently with Ctrl+S.
  • Use descriptive project names (e.g., "Acme Spring Sale 2025" rather than "test").
  • Back up the data/ directory regularly if you're self-hosting.
  • Save custom templates for layouts you reuse often across campaigns.
